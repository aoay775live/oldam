# Правильная Шарманка на радиолампе 6П3С от дяди Артемия

Фан-проект на **GNU Radio Companion**. Граф `oldam.grc` имитирует АМ-модуляцию
самодельного лампового передатчика «из говна и палок» — такие радиохулиганы
собирают и вещают на ~3 МГц. Цель не чистая модуляция, а максимально **грязный,
аутентичный звук дедовской АМ-шарманки**: плывущая несущая, хрипящий перегруз,
паразитная ЧМ, писк самовозбуждения, фон 50 Гц от кривой сетевой фильтрации.

На выходе — комплексный I/Q сигнал, который скармливается в SDR-приёмник
(HDSDR и т.п.) как «soundcard input», чтобы слушать результат как настоящий
эфир.

## Что нужно

- **GNU Radio 3.10.12** (ставился через [radioconda](https://github.com/ryanvolz/radioconda)).
- Два виртуальных аудиокабеля (VAC):
  - **VoiceMeeter** (VB-Audio) — отдаёт звук-источник в граф.
  - **VB-CABLE** (VB-Audio Virtual Cable) — забирает I/Q с графа в SDR-приёмник.
- SDR-приёмник с режимом «soundcard / I/Q input» (HDSDR, SDR#, и т.д.).

> Используется Python именно из radioconda, не системный:
> `%userprofile%/radioconda/python.exe`

## Аудио-тракт

```
  [плеер/микрофон]                                   [HDSDR / SDR]
        │                                                  ▲
        ▼                                                  │
  VoiceMeeter Output  ──►  audio_source_0                  │
                              │                            │
                       [ грязная АМ-обработка ]            │
                              │                            │
                          audio_sink_0  ──►  CABLE Input ──┘
                          (2 канала = I и Q)
```

- Вход: `audio_source_0` ← **VoiceMeeter Output**.
- Выход: `audio_sink_0` → **CABLE Input**, 2 канала = I и Q.
- `samp_rate = 48000`. Несущая по умолчанию 1000 Гц (это центр в SDR, не реальные 3 МГц — проект про «звук», а не про РЧ-вещание).

Вход и выход — два **разных** устройства. В HDSDR выбираешь `CABLE Output`
как вход звуковой карты и видишь модулированный сигнал.

## Как запустить

Открыть граф в GUI:

```
gnuradio-companion oldam.grc
```

и нажать ▶ (Run). Откроется окно «AM Modulator Professional» с ползунками,
осциллографами и водопадом.

Проверить/скомпилировать граф без запуска GUI:

```
%userprofile%/radioconda/python.exe %userprofile%/radioconda/Scripts/grcc-script.py -o <outdir> oldam.grc
```

## Ручки в GUI

| Ручка | Что делает |
|---|---|
| **Carrier Frequency** | Частота несущей (центр в SDR). |
| **Modulation Depth** | Глубина АМ (до 150% — намеренный перемод). |
| **Negative Peak Limit** | Ограничение отрицательного пика огибающей. |
| **Output Power** | Уровень выходного сигнала. |
| **HF Boost** | Подъём верхов в звуке. |
| **Audio Pull** | Увод несущей от громкости звука («подсадка» VFO). |
| **Drift** | Медленный случайный дрейф несущей. |
| **Parasitic FM** | Быстрая паразитная частотная модуляция. |
| **Self-Osc (Level / ON)** | Самовозбуждение — писк, бегающий по частоте. |
| **Hum 50Hz (Level / ON / Dynamic)** | Сетевой фон 50 Гц с гармониками; Dynamic — реакция фона на громкость. |
| **Antenna Effect** | «Качание» антенны — медленный ЧМ-увод. |
| **Splatter Drive** | Перегрузка/сплаттер (мягкий клиппинг tanh). |
| **Audio Profile** | Профиль звука: Music / Mic. |
| **Waterfall Brightness** | Яркость водопада (визуал). |

Есть ещё экспериментальные ручки (**Mic Chaos**, **X-Mode**) — частично рабочие,
часть веток в графе сведена в «кладбище» и на выход не влияет.

## Сигнальная цепочка (вкратце)

1. Звук чистится и формируется: LPF, HF-boost, профиль Music/Mic.
2. Жёсткая обработка: асимметричный клиппинг, soft-clip (tanh), двойной AGC.
3. Подмешиваются паразиты: самовозбуждение, фон 50 Гц с гармониками.
4. Формируется мгновенная частота несущей: несущая + audio pull + дрейф +
   паразитная ЧМ + antenna wobble → «грязный VFO».
5. АМ: огибающая умножается на комплексную несущую.
6. Выход: power → split I/Q → hard-clip каждого канала → `audio_sink_0`
   + осциллограф + водопад.

## Состояние

Рабочий фан-проект. Известные баги пофикшены (в т.ч. детерминированный обрыв
сигнала на 80-й секунде из-за Inf от импульсного источника шума — см.
`CLAUDE.md`). Часть экспериментальных механик не доведена до конца.

- `backups/` — ручные снапшоты графа.
- `CLAUDE.md` — внутренние заметки: окружение, разбор багов, отладка.

Авторство исходной генерации указано как «Артём Косицын» в параметрах графа.
